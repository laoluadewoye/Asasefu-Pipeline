Instructions

Website Layout

This section is the Instructions section. This section will tell you how to use the website to download things and navigate the page.

After this section, you will see the Set Parsing Details section. This section will have various configuration options and a submission button to send the results to the backend service.

When you start a parse, a Parsing Progress will temporary appear inside the Set Parsing Details section below the configurable settings. This will show real-time parsing information so you don't get bored if you are waiting on a particularly long or popular story.

Finally, after the Set Parsing Details section is the Completed Parses section. This section will be where you can view your parsed information and filter the results based on the ID or the nickname you give the parsing session.

Buttons at the Top

Firstly, there are three buttons at the top of the page. These just lead my GitHub account and the pages where the overall project and this specific part of it is kept.

Working with the Parsing Settings

There are several settings that can be modified to customize the look and scope of your parsing. However, only two of them are required to actually parse a chapter or story.

  1. The first required setting is the Parsing Method. They will be the first two buttons in the Set Parsing Details section and be labeled Chapter and Story. You can only choose one, but you MUST choose one.
  2. The second required setting is the Chapter or Story Link. This will be the first text box in the section and it is here you would enter in your story and chapter link, but it must follow certain rules.
    1. Links that don't start like "https://archiveofourown.org/works/" or "https://archiveofourown.org/works/XXXXXXXX/chapters/" where "XXXXXXXX" represents a story id are automatically rejected. This is ONLY a story parser to stop people from just running wild on the site.
    2. "https://archiveofourown.org/works/XXXXXXXX" where "XXXXXXXX" represents a story id is a valid link if you want to parse either a chapter or story.
    3. "https://archiveofourown.org/works/XXXXXXXX/chapters/YYYYYYYYY where "XXXXXXXX" represents a story id and "YYYYYYYYY" represents a chapter id is ONLY a valid link for parsing a chapter. If you choose story and then pass this link, you will get an error notification.
    4. You can pass any chapter from a story into the chapter parser. You are not limited just getting chapter one from your 100-chapter story.
  3. The rest of the settings are optional. The third setting is the Nickname setting. You can give your parsing session a nickname so it's easier to find in the Completed Parses section.
  4. The last group of settings is kept under the Configure Advanced Settings section. These settings control the level of detail you want your parse to have. By default, the level of detail is capped so that you don't die before your parse is finished. That being said, if you really want to get everything, set the corresponding value for each advanced setting to -1.
    1. Max Comment Thread Depth controls how deep you want to search into comment threads. By default, this option is set to 10 replies. As an example, if a comment has 15 reply threads, but one of the reply threads has a back-and-forth that goes on for 20 replies, then that 20-reply conversation is cut off for brevity.
    2. Max Comment Page Limit controls the maximum amount of pages of comments you want to search per chapter. If a chapter has less pages than is specified here, then all the chapter's comments will be parsed. By default, this option is set to 3 pages.
    3. Max Kudos Page Limit controls the maximum amount of pages of kudos you want to search per story. If a story has less pages than is specified here, then all the story's kudos will be parsed. By default, this option is set to 3 pages.
    4. Max Bookmarks Page Limit controls the maximum amount of pages of bookmarks you want to search per story. If a story has less pages than is specified here, then all the story's bookmarks will be parsed. By default, this option is set to 3 pages.

Submitting and Waiting for your Story/Chapter

Once you hit the Start Parsing button, the web application will then start the parsing process. A real time chapter-by-chapter progress text will display until 1) either the parsing is complete, 2) or the app experience an error and has to end early, or 3) you cancel the in-progress parsing early.

Once that's done, you should see the parsed content in the Completed Parses section. You will be able to navigate through all the data and choose which data you want to save to JSON, XML, and YAML formats.

Navigating the Completed Parses Section

Once a parse its completed, you should see a green or yellow notification letting you know that. It is then you can go to the Completed Parses section to view the results. This section is represented with a hollow red-outlined rectangle.

At very top of this box is a search bar where you can use the session ID or nickname of your parsing session to filter which stories or chapters you want to look at. Below that, you would see a Refresh View button that you can use to refresh the result contents in case what you were expecting hadn't shown up yet.

In the sections themselves, you press one of the tabs to open the dropdown of a specific result type. To expand or close more drop downs, or to download things, you use the buttons that will be located on drop downs with a complex array of content.
